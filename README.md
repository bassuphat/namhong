# นามหงส์ กรุ๊ป — Corporate Website

เว็บไซต์ต้นแบบระดับ Enterprise สำหรับกลุ่มบริษัทนามหงส์ (บริษัท นามหงส์น้ำมันปาล์ม จำกัด และ บริษัท นามหงส์ พาวเวอร์ จำกัด) พัฒนาเป็น **static HTML/CSS/JavaScript** ล้วน ไม่มี build step ไม่มี dependency ภายนอก จึงเปิดดูได้ทันทีและนำไปวางบน hosting ใดก็ได้ หรือฝังเข้า backend/CMS ภายหลังได้ง่าย

## โครงสร้างไฟล์

```
namhong-website/
├── index.html                 หน้าแรก
├── about.html                 เกี่ยวกับเรา (ประวัติ/ไทม์ไลน์/วิสัยทัศน์/ผู้บริหาร/โครงสร้างองค์กร/ธรรมาภิบาล)
├── business.html              ภาพรวมธุรกิจ (หน้ากลาง เชื่อม 2 บริษัท)
├── business-palm-oil.html     นามหงส์ น้ำมันปาล์ม
├── business-power.html        นามหงส์ พาวเวอร์
├── sustainability.html        ความยั่งยืน (ESG + วงจรเศรษฐกิจหมุนเวียน)
├── news.html                  ข่าวสาร (มีตัวกรองหมวดหมู่)
├── career.html                ร่วมงานกับเรา (สวัสดิการ/ตำแหน่งงาน/ฟอร์มสมัคร)
├── contact.html                ติดต่อเรา (ฟอร์ม/แผนที่/ที่ตั้ง)
├── privacy-policy.html        นโยบายความเป็นส่วนตัว & PDPA
├── sitemap.xml / robots.txt
├── DEPLOY-TO-HOSTATOM.md      วิธีติดตั้งบน Hostatom ครบทุกขั้นตอน ตั้งแต่สมัครจนเว็บขึ้นจริง (เริ่มอ่านที่นี่)
├── BACKEND-SETUP.md           เอกสารอ้างอิงทางเทคนิคของระบบหลังบ้าน (มีอะไรป้องกันไว้บ้าง/แก้ปัญหาเบื้องต้น)
├── backend/                   ระบบหลังบ้านจริง: รับฟอร์ม → เก็บ MySQL → แดชบอร์ดผู้ดูแลระบบ
└── assets/
    ├── css/style.css          ระบบดีไซน์ทั้งหมด (tokens + components)
    └── js/main.js             ระบบ interactivity ทั้งหมด
```

ทุกหน้าใช้ไฟล์ CSS/JS ชุดเดียวกัน — แก้สีหรือพฤติกรรมที่ไฟล์เดียวมีผลทั้งเว็บไซต์

## ระบบดีไซน์

- **สี**: Primary Blue `#003B8E`, White, Silver/Light-Gray, Dark Navy text ตามที่กำหนด บวกสีรองสำหรับแยกสองธุรกิจ (Palm = amber, Power = teal) ใช้อย่างประหยัดเพื่อช่วยนำทางเท่านั้น
- **ฟอนต์**: Prompt (หัวข้อ), Noto Sans Thai (เนื้อหา), Inter (ตัวเลข/UI/ภาษาอังกฤษ) โหลดจาก Google Fonts
- **ลายเซ็นของเว็บไซต์ (signature element)**: แผนภาพ "flow diagram" แสดงวงจร ปาล์ม → โรงงานสกัด → น้ำมัน/ของเสีย → ก๊าซชีวภาพ → โรงไฟฟ้า ปรากฏแบบจางๆ ในพื้นหลัง Hero และแบบเต็มในหน้าความยั่งยืน/หน้าธุรกิจ — สื่อสารจุดขายที่แท้จริงของกลุ่มบริษัท คือธุรกิจสองอย่างที่เกื้อกูลกัน
- **การเข้าถึงง่าย**: ขนาดตัวอักษรพื้นฐาน 17px, line-height กว้าง, ปุ่ม skip-to-content, โฟกัสชัดเจน, aria-label ครบ, รองรับ `prefers-reduced-motion`

## ⚠️ ข้อมูลที่เป็น "ตัวอย่าง" กรุณาแก้ไขก่อนเผยแพร่จริง

ตามที่ระบุไว้ในบรีฟ — **ไม่มีการเดาข้อมูลบริษัท** ทุกจุดที่เป็นตัวอย่างจะมีกล่องข้อความสีอำพัน "กรุณาใส่ข้อมูลจริง" กำกับไว้ในหน้าเว็บเอง จุดสำคัญที่ต้องแก้ไข:

| จุดที่ต้องแก้ | อยู่ที่ไหน |
|---|---|
| เบอร์โทร/อีเมล/ที่อยู่จริง | Top bar และ Footer ทุกหน้า, contact.html |
| พิกัด Google Maps จริง | contact.html (มีคอมเมนต์วิธีฝัง iframe ไว้ให้) |
| ตัวเลขกำลังการผลิต, จำนวนพนักงาน, กำลังไฟฟ้า | index.html, business-palm-oil.html, business-power.html |
| รายชื่อและตำแหน่งผู้บริหารจริง + รูปภาพ | about.html |
| ปีในไทม์ไลน์ประวัติบริษัท | about.html |
| ใบรับรองมาตรฐานที่มีจริง (GMP/HACCP/RSPO/ISO ฯลฯ) | index.html, business-palm-oil.html |
| ข่าว/ตำแหน่งงานจริง | news.html, career.html |
| รูปภาพจริงทั้งหมด | ทุกจุดที่เป็นกล่อง "ภาพตัวอย่าง" (ดูหัวข้อรูปภาพด้านล่าง) |
| นโยบายความเป็นส่วนตัวฉบับเต็ม | privacy-policy.html — **ต้องให้ที่ปรึกษากฎหมายตรวจสอบ ไม่ใช่เอกสารกฎหมายที่ใช้ได้ทันที** |

**ข้อมูลที่อ้างอิงจากแหล่งสาธารณะจริง** (ไม่ใช่การเดา แต่ควรตรวจทานความถูกต้องล่าสุดอีกครั้งก่อนเผยแพร่): ปีก่อตั้ง พ.ศ. 2545, ที่ตั้งโรงงานอำเภอเขาพนม จังหวัดกระบี่, ผลิตภัณฑ์หลักคือน้ำมันปาล์มดิบและเมล็ดในปาล์มอบแห้ง, และระบบก๊าซชีวภาพ Hybrid Cover Lagoon ที่พัฒนาร่วมกับสถาบันวิจัยและพัฒนาพลังงานนครพิงค์ มหาวิทยาลัยเชียงใหม่

## รูปภาพ

ตามคำสั่ง "ห้ามเดา" เว็บไซต์นี้ **ไม่ได้ใส่ภาพถ่ายจริงหรือภาพสต็อกที่แอบอ้างว่าเป็นโรงงาน/บุคลากรของบริษัท** เพราะจะเป็นการให้ข้อมูลเท็จ แทนที่ด้วยกล่อง placeholder ที่ออกแบบสวยงาม พร้อมไอคอนและคำบรรยายว่าใครควรใส่ภาพอะไร (เช่น "ภาพโรงงานสกัดน้ำมันปาล์ม จังหวัดกระบี่ — กรุณาใส่ภาพจริง") เพียงแทนที่ `<div class="img-ph ...">...</div>` ด้วย `<img>` จริงเมื่อได้รับภาพถ่ายจากบริษัท

## หมายเหตุด้านเทคนิคที่ควรทราบ

1. **ภาษา TH/EN**: ปุ่มสลับภาษาทำงานได้จริงสำหรับเมนู/ฟุตเตอร์/ปุ่มทั่วเว็บไซต์ และเนื้อหาทั้งหมดในหน้าแรก ส่วนเนื้อหาในหน้าอื่นยังเป็นภาษาไทยเป็นหลัก (ระบบ i18n ที่วางไว้รองรับการแปลเพิ่มเติมได้ทันทีผ่าน `data-i18n="key"` ใน HTML และ dictionary ใน `assets/js/main.js`) นอกจากนี้การเลือกภาษาจะรีเซ็ตเมื่อเปลี่ยนหน้า เนื่องจากตั้งใจไม่ใช้ localStorage/cookie ในเวอร์ชันนี้ — เมื่อขึ้นระบบจริงหากต้องการให้จำภาษาไว้ข้ามหน้า แนะนำให้ผูกกับคุกกี้ (ควรตรวจสอบข้อกำหนด PDPA เรื่องคุกกี้อีกครั้งก่อนทำ เนื่องจากตอนนี้เว็บไซต์ไม่มีแบนเนอร์ขอความยินยอมแล้ว)
2. **CMS/Backend**: บรีฟระบุให้โครงสร้างรองรับ PHP/Laravel/MySQL ในอนาคต เนื่องจากไฟล์นี้เป็น static frontend ล้วน จึงยังไม่ได้ประกอบ backend จริง แต่โครงสร้างถูกออกแบบให้ย้ายเข้า Laravel ได้ง่าย: วาง `assets/` ใน `public/`, แปลงแต่ละหน้าเป็น Blade view, และให้ส่วนข่าว/ตำแหน่งงาน/แกลเลอรี/แบนเนอร์ดึงจากฐานข้อมูลแทนเนื้อหาตัวอย่าง
3. **Security (CSRF/XSS/SQLi/Rate limit/Captcha)**: เป็นเรื่องของ backend ที่ยังไม่มีในเวอร์ชันนี้ — ต้องทำเมื่อประกอบ Laravel backend จริง
4. **Lighthouse 95+**: โค้ดถูกเขียนตามแนวทาง performance/SEO/accessibility ที่ดี (semantic HTML, ไม่มีภาพหนักๆ, font-display swap, defer script) แต่คะแนนจริงขึ้นกับ hosting/CDN/ภาพที่ใส่จริงด้วย แนะนำให้รัน Lighthouse ซ้ำหลังใส่เนื้อหา/รูปภาพจริงและ deploy แล้ว

## Deploy ขึ้น Hostatom + ระบบหลังบ้าน PHP/MySQL

เปลี่ยนจากแผน Netlify เดิมมาใช้ **Hostatom** (เว็บโฮสติ้งไทยแบบเช่าจริง) พร้อมระบบหลังบ้าน PHP + MySQL ของตัวเอง ตามที่ต้องการเก็บข้อมูลฟอร์มไว้ดูเองได้

**สรุปขั้นตอน** (รายละเอียดทีละขั้นตอน พร้อมชื่อเมนูจริงของ Hostatom อยู่ใน **DEPLOY-TO-HOSTATOM.md**):
1. อัปโหลดไฟล์ทั้งหมด (รวมโฟลเดอร์ `backend/`) ขึ้น `httpdocs/` ผ่าน Plesk File Manager (ชื่อโฟลเดอร์คนละชื่อกับ `public_html` ที่ cPanel ใช้ — Hostatom ใช้ Plesk เป็นหลัก รายละเอียดดู DEPLOY-TO-HOSTATOM.md)
2. สร้างฐานข้อมูล MySQL + รันไฟล์ `backend/schema.sql` ผ่าน phpMyAdmin
3. แก้ `backend/config.php` ใส่ชื่อฐานข้อมูล/ผู้ใช้/รหัสผ่านจริง
4. เปิด `backend/admin/create-admin.php` ครั้งเดียวเพื่อสร้างบัญชีผู้ดูแลระบบ แล้วลบไฟล์นั้นทิ้ง
5. เข้าดูข้อความ/ใบสมัครงานที่ `backend/admin/login.php`
6. อย่าลืม find-and-replace `namhonggroup.co.th` (canonical, OG tags, sitemap.xml, robots.txt, schema.org) ให้เป็นโดเมนจริงที่ซื้อ

ฟอร์มติดต่อเราและฟอร์มสมัครงานต่อกับ `backend/contact-handler.php` และ `backend/apply-handler.php` ไว้แล้ว (ไม่ใช่ Netlify Forms อีกต่อไป) ทดสอบจริงแล้วว่าทำงานถูกต้อง ทั้งการบันทึกลง MySQL, ป้องกัน SQL Injection/XSS, กันสแปมเบื้องต้น (honeypot + จำกัดความถี่ต่อ IP), และแดชบอร์ดผู้ดูแลระบบที่มี CSRF protection — ดูรายละเอียดการป้องกันทั้งหมดใน BACKEND-SETUP.md

## การพรีวิว

เปิด `index.html` ด้วยเบราว์เซอร์ได้ทันทีสำหรับดูหน้าตาเว็บไซต์ ส่วนฟอร์มติดต่อเรา/สมัครงานต้องมี PHP + MySQL รันอยู่จริงถึงจะทดสอบได้ครบ (ทดสอบบนเครื่อง dev ได้ด้วย PHP built-in server เช่น `php -S localhost:8000` ในโฟลเดอร์ `backend/` หลังตั้งค่าฐานข้อมูลแล้ว หรือรอทดสอบบน Hostatom จริงหลัง deploy)
