# ระบบหลังบ้าน (PHP + MySQL) — เอกสารอ้างอิงทางเทคนิค

ระบบนี้รับข้อมูลจากฟอร์ม "ติดต่อเรา" และ "สมัครงาน" เก็บลงฐานข้อมูล MySQL จริง พร้อมหน้าแดชบอร์ดผู้ดูแลระบบ (ต้องล็อกอิน) สำหรับดู/จัดการข้อมูลที่ส่งเข้ามา

> **ขั้นตอนติดตั้งแบบทีละคลิกทั้งหมด (สมัคร Hostatom → อัปโหลด → สร้าง DB → SSL → ใช้งานจริง) ย้ายไปอยู่ที่ไฟล์ `DEPLOY-TO-HOSTATOM.md` แล้ว** ไฟล์นี้เหลือไว้เป็นเอกสารอ้างอิงเฉพาะเรื่องระบบหลังบ้าน (มีอะไรป้องกันไว้บ้าง / แก้ปัญหาเบื้องต้น) — ถ้าเพิ่งเริ่มติดตั้ง ให้ไปที่ `DEPLOY-TO-HOSTATOM.md` ก่อน

**ทดสอบแล้วจริงก่อนส่งมอบ:** รันทั้งระบบ (PHP 8.3 + MariaDB) และยิงคำขอจริงทดสอบทุกเส้นทาง — ส่งฟอร์มสำเร็จ/ฟอร์มข้อมูลผิด/บอทติดกับดัก/ป้องกันสแปมด้วยการจำกัดความถี่/ป้องกัน SQL Injection/ป้องกัน XSS/ระบบล็อกอิน-CSRF ของแดชบอร์ด/ปุ่มทำเครื่องหมายอ่านแล้ว-ลบ ทุกอย่างผ่านหมด รวมถึงเจอบั๊ก 1 จุดระหว่างทดสอบ (ฟังก์ชันขาดหายในหน้าตั้งค่าแอดมินครั้งแรก) และแก้ไขเรียบร้อยแล้วก่อนส่งไฟล์นี้ให้

## โครงสร้างไฟล์ backend/

```
backend/
├── config.php          ตั้งค่าฐานข้อมูล (แก้ 3 บรรทัดตาม DEPLOY-TO-HOSTATOM.md ขั้นตอนที่ 6)
├── db.php              ตัวเชื่อมต่อฐานข้อมูล + ตัวจำกัดความถี่การส่งฟอร์ม
├── contact-handler.php รับข้อมูลจากฟอร์มติดต่อเรา
├── apply-handler.php   รับข้อมูลจากฟอร์มสมัครงาน
├── schema.sql          คำสั่งสร้างตารางฐานข้อมูล (รันครั้งเดียวผ่าน phpMyAdmin)
└── admin/
    ├── create-admin.php   สร้างบัญชีผู้ดูแลระบบครั้งแรก (ลบทิ้งหลังใช้!)
    ├── login.php / logout.php
    ├── dashboard.php       ดู/จัดการข้อความและใบสมัครงาน
    └── auth.php, style.css
```

## มีอะไรป้องกันไว้ให้แล้วบ้าง

- **SQL Injection** — ใช้ Prepared Statement (PDO) ทุกจุด ไม่มีการต่อ string SQL จากข้อมูลผู้ใช้เลย
- **XSS** — ข้อมูลที่แสดงในแดชบอร์ดผ่าน `htmlspecialchars()` ทุกช่อง ป้องกันสคริปต์แอบแฝงจากผู้ส่งฟอร์ม
- **CSRF** — ปุ่ม "อ่านแล้ว/ลบ" ในแดชบอร์ดมี token ป้องกัน ถ้า token ผิดจะไม่มีผลใดๆ
- **สแปม/บอทเบื้องต้น** — Honeypot field (ช่องที่บอทกรอกแต่คนมองไม่เห็น) + จำกัดจำนวนครั้งต่อ IP (ค่าเริ่มต้น 5 ครั้ง/10 นาที ปรับได้ที่ `config.php`)
- **รหัสผ่านผู้ดูแลระบบ** — เก็บแบบ hash (`password_hash`) ไม่เก็บรหัสผ่านตรงๆ ในฐานข้อมูล
- **Session** — ตั้งค่า HttpOnly cookie + regenerate session ID ตอนล็อกอินสำเร็จ กันการขโมย session
- **create-admin.php ล็อกตัวเองอัตโนมัติ** — ทำงานได้แค่ตอนที่ยังไม่มีบัญชีผู้ดูแลระบบเลย ป้องกันคนอื่นมาสร้างบัญชีแอบเพิ่มทีหลัง แม้จะลืมลบไฟล์ทิ้ง (แต่ก็ยังควรลบทิ้งอยู่ดี เป็นความปลอดภัยอีกชั้น)

## ยังไม่ได้ทำ / แนะนำเพิ่มเติมภายหลัง

- **Captcha แบบเต็มรูปแบบ** (เช่น Google reCAPTCHA) — ตอนนี้ป้องกันด้วย Honeypot + Rate limit ซึ่งกันบอทพื้นฐานได้ดี ถ้าสแปมยังเยอะในอนาคต แจ้งได้ จะช่วยต่อ reCAPTCHA ให้ (ต้องสมัคร API key ฟรีจาก Google ก่อน)
- **สำรองข้อมูล** — แนะนำตั้งค่า Backup ฐานข้อมูลอัตโนมัติใน Plesk เป็นระยะ (Plesk มีเครื่องมือ Backup Manager ในตัว)

## แก้ปัญหาเบื้องต้น

- **ฟอร์มขึ้น "ระบบขัดข้อง"** — มักเกิดจาก `config.php` ยังไม่ได้แก้เป็นค่าจริง หรือยังไม่ได้รัน `schema.sql`
- **หน้า login/dashboard ขาว/error พัง** — ตรวจสอบว่า Hostatom เปิด PHP เวอร์ชัน 8.0 ขึ้นไป และมี extension `mbstring`, `pdo_mysql` เปิดใช้งาน (ปกติเปิดเป็นค่าเริ่มต้นอยู่แล้วในแพ็กเกจ Hostatom ทั่วไป — ถ้าไม่แน่ใจ เช็คได้ที่ Plesk → PHP Settings)
- **ล็อกอินแอดมินไม่ได้** — ถ้าลืมรหัสผ่าน ต้องเข้า phpMyAdmin → ตาราง `admin_users` → ลบแถวเดิมทิ้ง แล้วเปิด `create-admin.php` ใหม่อีกครั้ง (จะกลับมาใช้งานได้เพราะตอนนี้ไม่มีบัญชีเหลืออยู่)
