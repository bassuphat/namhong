# วิธีติดตั้งเว็บไซต์บน Hostatom (ครบทุกขั้นตอน ตั้งแต่สมัครจนเว็บขึ้นจริง)

> คู่มือนี้อ้างอิงจากข้อมูลจริงของ Hostatom ที่เช็คมาให้แล้ว **Hostatom ใช้ Control Panel สองแบบให้เลือกตอนซื้อแพ็กเกจ คือ Plesk หรือ DirectAdmin** — คู่มือนี้เขียนตาม **Plesk** เป็นหลัก เพราะเป็นระบบที่ Hostatom แนะนำและใหม่กว่า (มีหมายเหตุสำหรับ DirectAdmin ไว้ท้ายไฟล์ ถ้าเลือกอันนั้นแทน)

---

## ขั้นตอนที่ 1 — สมัครแพ็กเกจ Hosting

1. เข้า [hostatom.com](https://www.hostatom.com) → เลือก **Web Hosting** (แพ็กเกจ Shared Hosting เริ่มต้นก็เพียงพอ เว็บไซต์นี้เป็น static HTML/CSS/JS + PHP เบาๆ ไม่กินทรัพยากรมาก)
2. ตอนเลือกแพ็กเกจจะมีตัวเลือก Control Panel — เลือก **Plesk**
3. ถ้ายังไม่มีโดเมน สามารถจดพร้อมกันได้ในหน้าเดียวกันเลย (เช่น `namhong.com` หรือ `namhonggroup.co.th`) ถ้ามีโดเมนอยู่แล้วจากที่อื่น เลือก "ใช้โดเมนที่มีอยู่แล้ว"
4. ชำระเงิน กรอกข้อมูลลูกค้าให้ครบ
5. รออีเมลยืนยันจาก Hostatom (มักไม่กี่นาทีถึงไม่กี่ชั่วโมง) จะได้ Username/Password สำหรับเข้า **portal.hostatom.com**

## ขั้นตอนที่ 2 — ถ้าโดเมนซื้อจากที่อื่น ต้องชี้มาที่ Hostatom

*(ถ้าจดโดเมนพร้อมแพ็กเกจในขั้นตอนที่ 1 แล้ว ข้ามไปขั้นตอนที่ 3 ได้เลย)*

1. เข้า portal.hostatom.com → เมนู **บริการของฉัน** → เปิดบริการที่ซื้อไว้ → ดู Nameserver ที่ต้องใช้ (มักเป็นรูปแบบ `ns1.hostatom.com`, `ns2.hostatom.com`)
2. เข้าไปที่ผู้ให้บริการจดโดเมนเดิม (เช่น GoDaddy, Namecheap, THNIC) → หาเมนู DNS/Nameserver → เปลี่ยนเป็น Nameserver ของ Hostatom
3. รอ DNS อัปเดตทั่วโลก (Propagate) ปกติ 2-24 ชั่วโมง

## ขั้นตอนที่ 3 — เข้า Plesk Control Panel

1. Log in ที่ portal.hostatom.com ด้วย Username/Password ที่ได้รับ
2. ไปที่เมนู **บริการ (Services)** → เลือกแพ็กเกจที่ซื้อไว้
3. กดปุ่ม **Login to Plesk Control Panel**

## ขั้นตอนที่ 4 — อัปโหลดไฟล์เว็บไซต์

1. ใน Plesk → **Websites & Domains** → เลือกโดเมนของคุณ → **File Manager**
2. โฟลเดอร์หลักของเว็บไซต์ใน Plesk ชื่อ **`httpdocs`** (คนละชื่อกับ `public_html` ที่ cPanel/DirectAdmin ใช้ — อย่าสับสน)
3. เข้าไปในโฟลเดอร์ `httpdocs` → กด **Upload** → เลือกไฟล์ `namhong-website.zip`
4. เมื่ออัปโหลดเสร็จ คลิกขวาที่ไฟล์ zip → **Extract** (แตกไฟล์) จะได้โฟลเดอร์ `namhong-website/` ข้างใน
5. **สำคัญ:** เข้าไปในโฟลเดอร์ `namhong-website/` → เลือกไฟล์/โฟลเดอร์ทั้งหมดข้างใน (Select All) → กด **Move** → ย้ายออกมาไว้ที่ `httpdocs/` โดยตรง
   - ถ้าข้ามขั้นตอนนี้ เว็บไซต์จะเปิดได้ที่ `yourdomain.com/namhong-website/` แทนที่จะเป็น `yourdomain.com/` ตรงๆ
6. ลบไฟล์ `namhong-website.zip` และโฟลเดอร์ `namhong-website/` ที่ว่างเปล่าทิ้ง (ไม่จำเป็นต้องเก็บไว้)
7. ตรวจสอบว่า `httpdocs/` มีไฟล์ `index.html`, `about.html`, โฟลเดอร์ `assets/`, โฟลเดอร์ `backend/` อยู่ตรงนั้นเลย

## ขั้นตอนที่ 5 — สร้างฐานข้อมูล MySQL

1. ใน Plesk → **Websites & Domains** → โดเมนของคุณ → แท็บ **Databases**
2. กด **Add Database** → ตั้งชื่อฐานข้อมูล เช่น `namhong`
3. ตั้งชื่อผู้ใช้ฐานข้อมูล + รหัสผ่านที่คาดเดายาก (Plesk จะสร้างผู้ใช้พร้อมสิทธิ์เข้าฐานข้อมูลนี้ให้อัตโนมัติ)
4. **จดชื่อฐานข้อมูลเต็ม + ชื่อผู้ใช้เต็ม + รหัสผ่านไว้ให้แม่นยำ** (Plesk มักเติม prefix ของ subscription นำหน้าชื่อที่ตั้งอัตโนมัติ ให้คัดลอกชื่อที่ระบบแสดงจริง ไม่ใช่ชื่อที่พิมพ์ตอนแรก)
5. ในหน้า Databases เดียวกัน กดปุ่ม **phpMyAdmin** ข้างฐานข้อมูลที่สร้าง
6. ในหน้า phpMyAdmin → แท็บ **SQL** → เปิดไฟล์ `backend/schema.sql` (อยู่ในไฟล์ที่อัปโหลดไปแล้ว) → คัดลอกเนื้อหาทั้งหมดวางในช่อง → กด **Go**
7. ตรวจว่าปรากฏตาราง 4 ตาราง: `contact_messages`, `job_applications`, `admin_users`, `submission_log`

## ขั้นตอนที่ 6 — แก้ไข config.php

กลับไปที่ **File Manager** → เปิดไฟล์ `httpdocs/backend/config.php` (คลิกขวา → Edit) แก้ 3 บรรทัดให้ตรงกับขั้นตอนที่ 5:

```php
define('DB_NAME', 'ชื่อฐานข้อมูลเต็มจาก Plesk');
define('DB_USER', 'ชื่อผู้ใช้เต็มจาก Plesk');
define('DB_PASS', 'รหัสผ่านจริงที่ตั้งไว้');
```

`DB_HOST` ไม่ต้องแก้ (`localhost` ใช้ได้เลย) บันทึกไฟล์ (Save)

## ขั้นตอนที่ 7 — เปิดใช้งาน SSL/HTTPS ฟรี

1. Plesk → **Websites & Domains** → โดเมนของคุณ → **Security** → **SSL/TLS Certificates**
2. กด **Install a free basic certificate**
3. เลือก **Let's Encrypt** → กด **Get it free** → กด **Reload**
4. รอสักครู่ ตรวจสอบสถานะว่าติดตั้งสำเร็จ (จะมี certificate ปรากฏในรายการ)

## ขั้นตอนที่ 8 — สร้างบัญชีผู้ดูแลระบบ (ทำครั้งเดียว แล้วลบไฟล์ทิ้ง)

1. เปิดเบราว์เซอร์ไปที่ `https://โดเมนของคุณ/backend/admin/create-admin.php`
2. ตั้งชื่อผู้ใช้และรหัสผ่านสำหรับเข้าดูข้อความ/ใบสมัครงาน (คนละอันกับรหัส MySQL ในขั้นตอนที่ 5)
3. เมื่อสร้างสำเร็จ กลับไปที่ File Manager → **ลบไฟล์ `httpdocs/backend/admin/create-admin.php` ทิ้งทันที**

## ขั้นตอนที่ 9 — แก้โดเมนใน SEO/Meta tags

โค้ดใช้ `namhonggroup.co.th` เป็นโดเมนตัวอย่างไว้ทั่วเว็บไซต์ (canonical tag, Open Graph, schema.org, sitemap.xml, robots.txt) — ถ้าโดเมนจริงไม่ใช่ชื่อนี้ ต้อง find-and-replace ให้เป็นโดเมนจริง บอกชื่อโดเมนที่ซื้อมาได้เลย จะช่วยแก้ให้ทุกจุดในทีเดียว

## ขั้นตอนที่ 10 — ทดสอบให้ครบก่อนประกาศใช้จริง

- [ ] เข้า `https://yourdomain.com` เช็คว่าหน้าเว็บขึ้นถูกต้อง มีรูปกุญแจ HTTPS สีเขียว/ปลอดภัย
- [ ] คลิกทดสอบเมนู/ลิงก์ทุกหน้า
- [ ] ทดสอบส่งฟอร์ม "ติดต่อเรา" ด้วยข้อมูลจริงของตัวเอง ดูว่าขึ้นข้อความสำเร็จ
- [ ] ทดสอบส่งฟอร์ม "สมัครงาน" เช่นกัน
- [ ] เข้า `https://yourdomain.com/backend/admin/login.php` ล็อกอินด้วยบัญชีที่สร้างไว้ ตรวจว่าเห็นข้อความ/ใบสมัครที่เพิ่งทดสอบส่งไป
- [ ] ทดสอบกดปุ่ม "อ่านแล้ว" และ "ลบ" ในแดชบอร์ด

## ถ้าเลือก DirectAdmin แทน Plesk ตอนสมัคร

หลักการเหมือนกันทั้งหมด มีจุดต่างดังนี้:

| เรื่อง | Plesk | DirectAdmin |
|---|---|---|
| โฟลเดอร์หลักเว็บไซต์ | `httpdocs` | `public_html` |
| เมนูสร้างฐานข้อมูล | Websites & Domains → Databases | MySQL Management |
| เมนู SSL | Security → SSL/TLS Certificates | SSL Certificates → Let's Encrypt |

ที่เหลือ (อัปโหลด/แตกไฟล์/แก้ config.php/สร้างบัญชีแอดมิน) ทำเหมือนกันทุกประการ เพียงสลับชื่อโฟลเดอร์และเมนูตามตารางด้านบน
